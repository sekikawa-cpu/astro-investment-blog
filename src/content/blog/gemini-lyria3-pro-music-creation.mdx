## GeminiのLyria 3 Proとは

**Lyria 3 Pro** は、Google DeepMindが開発した音楽生成AIモデルです。Geminiのインターフェース上で「音楽を作成」メニューから利用でき、テキストで楽曲の雰囲気や楽器、ジャンルを指定するだけで音楽ファイルを生成してくれます。ボーカル付きの楽曲にも対応しており、歌詞を渡せばそのまま歌にしてくれます。

---

## 実験①：最初のプロンプト試行

最初は「どんなプロンプトを書けばいいかわからない」という状態だったので、まずGeminiにプロンプトを考えてもらおうと思いました。

**プロンプト**

```
Geminiに音楽を作成してほしいです。以下の用語からプロンプトを出力してください。
メロウ
男性3人組のシルキーボイスによる華麗なハーモニー
サックスあり
夜に恋人とドライブのムードに最適
```

「プロンプトを出力してください」と書いたつもりだったのですが、Geminiはプロンプトを考える代わりに、**そのまま楽曲を生成してしまいました**。意図とは違う動作でしたが、結果として曲が出てきたのでそのまま聴いてみました。

<MusicPlayer src="/music/joseki-no-neiki.mp4" title="助手席の寝息" caption="初回生成楽曲" />

夜のドライブらしい落ち着いたトラックでしたが、シルキーボイスに期待していた甘さが少し足りない印象でした。「やや普通な声だな」というのが正直な感想です。指示が曖昧だと、AIもそれなりの出力をするということを実感した瞬間でした。

---

## 実験②：参照アーティストを具体的に伝える

「シルキーボイス」という言葉だけでは伝わらないと感じ、もっと具体的なイメージを渡すことにしました。

参考にしたのは、MIHIRO〜マイロ〜さんのYouTube動画です。あの甘くてメロウな歌声のスタイルを、GeminiにURLで伝えてみました。

**プロンプト**

```
ボーカルは以下のURLの男性でお願いします。
https://www.youtube.com/watch?v=YLqa7s6zhmM
```

**Geminiの出力メッセージ**

> MIHIRO 〜マイロ〜 さんのような、メロウでシルキーなボーカルをイメージして作成しました。サックスの音色が響く、夜のドライブにぴったりの曲をお楽しみください。

<MusicPlayer src="/music/joseki-tsukiakari-maro.mp4" title="助手席の月明かり" caption="MIHIRO〜マイロ〜スタイル参照版" />

聴いた瞬間、「これだ」と思いました。甘くてメロウな声質、サックスの絡み方、全体の雰囲気——私の望んだとおりの仕上がりになっていました。

抽象的なワードだけでなく、**具体的な参照（URL・アーティスト名）を渡す**ことで、Geminiのアウトプット精度が格段に上がることを実感しました。

## Geminiに音楽生成プロンプトの書き方を聞いてみた

2曲を作ってみて、プロンプトの書き方に興味が出てきたので、Gemini自身に聞いてみました。

**プロンプト**

```
Geminiに音楽を作成してほしいときのプロンプトを教えてください。
```

**Geminiの回答（要約）**

> **音楽生成プロンプトの基本要素**
>
> プロンプトには、以下の要素を自由に組み合わせてみてください。
>
> - **ジャンル**: Lo-Fi、アンビエント、クラシック、ジャズ、エレクトロニックなど
> - **雰囲気（ムード）**: 落ち着いた、先進的な、明るい、調和のとれた、前向きなど
> - **使用する楽器**: ピアノ、アコースティックギター、シンセサイザー、ストリングスなど
> - **テンポ**: ゆったりめ、アップテンポ、リズミカルなど
> - **用途・シチュエーション**: 勉強用、プレゼン用、リラックス用など

具体例として3つのプロンプトも提示してくれました。

**例①：資格試験の学習に集中したいとき**
```
DXビジネス検定の模擬試験を解く際に、集中力を高めるための歌詞のないLo-Fiヒップホップを
作成してください。テンポはゆったりめで、ピアノと環境音（カフェの環境音や雨の音など）を
混ぜた、落ち着いた雰囲気にしてください。
```

**例②：業務やプレゼン用のBGMを作りたいとき**
```
AI/DX推進の社内向けプレゼンテーションで、スライドの背景として流すBGMを作成してください。
シンセサイザーを使用したエレクトロニックなジャンルで、先進的で未来的な印象を与える、
明るすぎない曲調を希望します。
```

**例③：心の調和を保ち、リラックスしたいとき**
```
1日の終わりに心の調和を取り戻し、リフレッシュするためのアンビエント音楽を作ってください。
アコースティックギターをメインにした、静かで温かみのあるメロディで、ボーカルはなしでお願いします。
```

さらにコツとして「**ボーカルの有無を明記すること**」と「**対話でブラッシュアップすること**」を挙げていました。一度生成した曲に対して「もっとテンポを速くして」「ピアノの音をもっと目立たせて」と追加指示を出すことで、理想に近づけていけるとのことです。

## 応用：歌詞を先に作ってから音楽生成する

最初に生成した楽曲の歌詞が「少しチープかな」という印象だったので、**先に歌詞をGeminiに作ってもらい、それを使って音楽を生成する**というアプローチを試してみました。

### 歌詞込みの音楽生成プロンプト

この歌詞をそのまま使い、上記の要件と合わせて音楽生成を依頼しました。

> ご指定の歌詞と要件をもとに、楽曲を生成しました。ゆったりとしたテンポのR&Bで、ピアノとサックスが響く大人の雰囲気のトラックに仕上がっています。4人のボーカルによるセクシーな歌い分けや、サビの美しいハーモニーなど、胸がキュンとするような恋人との夜のドライブにぴったりの曲です。

## 投稿者の所感

AI/DX推進に携わる立場から、今回の体験を通じて気づいたことをまとめます。

**① プロンプトの「言語化力」が音楽品質を直接左右する**

最初の曲と最後の曲では、完成度に大きな差がありました。その差はAIの能力ではなく、**指示の解像度**によるものです。抽象的な「シルキーボイス」という言葉だけでは伝わらなかったものが、参照URLや具体的な役割分担（シルキー・タイニー・低音・コーラス）を渡すことで一気に具体化しました。これはコードの指示でも、業務フローの自動化指示でも、まったく同じ話です。

**② 「先に構造化する」アプローチは音楽生成にも通じる**

歌詞を先に作ってから音楽を生成するアプローチは、エージェントへの指示設計と同じ思考プロセスです。いきなり「音楽を作って」と投げるのではなく、要件を分解して構造化し、段階的にアウトプットを積み上げる。この発想は、AIを使ったあらゆる業務に応用できます。

**③ Apple Musicへの依存が少し薄れそうな予感**

今まで「Chill系プレイリスト」を聴き続けていましたが、自分の気分やシチュエーションに完全に合わせた楽曲を生成できるとなると、話が変わってきます。夜のドライブ用、集中作業用、朝のルーティン用——それぞれに最適なオリジナル楽曲を持てる時代が、もうそこまで来ています。

## まとめ

- GeminiのUIに「音楽を作成」機能が追加され、**Lyria 3 Pro**が楽曲生成を担う
- テキストプロンプトだけで楽曲が生成できるが、**指示の解像度がクオリティを直接左右する**
- 参照アーティストのURLを渡すことで、ボーカルスタイルをより精度高く伝えられる
- 事前に**歌詞を構造化してから音楽生成する**アプローチが完成度を大きく引き上げる
- プロンプト設計の考え方はAI活用全般に通じる汎用スキルである
- 自分好みのオリジナル楽曲を自作できる時代が本格的に到来しつつある
